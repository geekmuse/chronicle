## Codebase Patterns
- `prop_assert_eq!` (proptest macro) does NOT support implicit `{var}` capture in its format string argument — use explicit positional form `"msg {}", var` instead.
- `prop_oneof!` boxes all branches internally, so mixing `&str` (regex) strategies with `prop_map` results in one block is fine; no manual `.boxed()` needed.
- `Just(String::from("..."))` is idiomatic for generating a single fixed string value in a proptest strategy.
- When building JSON for proptest round-trips, prefer `serde_json::json!({...})` + `to_string` over raw `format!` strings — it handles escaping of special chars (e.g. `"` in XML attributes) transparently.
- Pre-commit hook (`prek`) stashes unstaged changes and runs the full test suite; a known lock-file race in `push_merges_new_entries_with_existing_repo_content` occasionally fires under parallel execution — passes in isolation.
- `serde_json` Object uses BTreeMap (alphabetical key order). Any fuzz target or proptest that compares JSON strings after parse→serialize must pre-normalise inputs or it will see spurious key-order failures.
- `serde_json` float serialisation is non-idempotent for some large numeric literals; double-normalise and skip non-idempotent inputs in fuzz targets.
- cargo-fuzz binary targets require `libfuzzer-sys` in `[dependencies]`, NOT `[dev-dependencies]`; fuzz workspace must have `[workspace]` + `.gitignore` covering `target/` and `artifacts/`.
