## Codebase Patterns
- **Status formatter pattern**: `StatusFormatter<W: io::Write>` takes a generic writer for testability. Tests pass `&mut Vec<u8>` to capture output. `ok/warn/err` are no-ops in porcelain mode; `kv` is a no-op outside porcelain mode. Color codes (ANSI_GREEN/YELLOW/RED/RESET constants) applied to the symbol only.
- **should_use_color helper**: `pub fn should_use_color(no_color_flag: bool) -> bool` — checks flag, then `NO_COLOR` env var, then `io::stdout().is_terminal()`. Marked `#[must_use]`.
- **Private inner write function**: `status_write<W: io::Write>(args, config_path, home, use_color, writer)` is private but accessible from `mod tests` (child modules can access parent private items in Rust). Allows output capture without spawning a process.
- **Config re-export**: `config::Config` is available directly (via `pub use schema::Config` in `config/mod.rs`). No need to spell out `config::schema::Config` in function signatures.
- **Atomic file write pattern**: serialize to `.<name>.<pid>.<nanos>.tmp` sibling, then `fs::rename` to target. Used by `StateCache`, `MaterializeCache`, and now `SyncState`. Always call `fs::create_dir_all(parent)` first.
- **Co-location convention**: state files sit as siblings of the repo dir (`<repo_path>/../<file>.json`). The `path_for_repo(repo_path)` method returns `repo_path.parent().unwrap_or(repo_path).join("<file>.json")`.
- **Non-fatal side-effect writes**: wrap state file writes in `if let Err(e) = ... { tracing::warn!(...) }` — never fail the main operation on bookkeeping errors.
- **Operation timing**: capture `std::time::Instant::now()` AFTER lock acquisition (before real work), use `.elapsed()` at end for accurate duration excluding jitter.
- **Module registration**: new modules must be added to `src/lib.rs` as `pub mod <name>;` so integration tests can import them.
