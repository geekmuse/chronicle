## Codebase Patterns
- `merge::entry`: `EntryKey` enum (`Header` | `Entry{entry_type,id}`); `parse_entry(raw)` returns `Option<ParsedEntry>`; `extract_timestamp(value)` checks `timestamp`, `created_at`, `createdAt` fields
- `merge::set_union::merge_jsonl(remote, remote_path, local, local_path, reporter)` â†’ `MergeOutput { content, malformed, conflicts }`; remote entries populate first (win on collision); prefix verification (Â§5.4) detects byte-inequivalent common entries
- `ConflictReporter` trait in `merge::set_union` â€” `report_prefix_mismatch(file, entry_key, local_raw, remote_raw)`; `NullReporter` is the no-op impl; real ring buffer impl comes in US-008
- `PrefixConflict { file, entry_key, local_raw, remote_raw }` â€” one record per divergent common entry; included in `MergeOutput::conflicts`
- Prefix verification loop: iterate local_entries; if key in remote map AND raws differ â†’ warn + reporter.report + push conflict; remote version stays in map (skip local); if key not in remote map â†’ insert local
- `MalformedLine { path, line_number, snippet }` â€” 1-based line, up to 80-char snippet; emitted via `tracing::warn!` and returned in `MergeOutput::malformed`
- Stable sort tie-break: `Source::Remote = 0`, `Source::Local = 1`, then `original_index` â€” guarantees deterministic output for equal timestamps
- `Path::new("x")` is NOT const-stable in current Rust edition; use `fn helper() -> &'static Path { Path::new("x") }` in test constants
- `#![allow(dead_code)]` needed on both `entry.rs` and `set_union.rs` until US-007/US-015 wire in the merge module
- `canonicalize_line(line, level)` / `decanonicalize_line(line)` on `TokenRegistry` â€” content-level L2/L3 dispatch in `src/canon/levels.rs`; returns `Result<String, ChronicleError>`
- `try_canonicalize_path(s)` â€” L2: returns `Some(canonical)` only if `s == $HOME` or `s.starts_with($HOME/)` (strict prefix, boundary `/` or EOS); custom tokens applied prefix-only with `break`
- `try_canonicalize_text(s)` â€” L3: `replace_in_text` scans globally; custom tokens also applied globally (no `break`)
- `try_decanonicalize_text(s)` â€” reverses both L2 and L3; always applied to ALL string values in de-canon (safe because token pattern unlikely to appear naturally)
- `replace_in_text(text, from, to)` â€” path-boundary replacement: `from` must be followed by `/` or EOS; prevents partial matches (e.g., `bradmatic2` won't match `bradmatic`); handles multiple occurrences in a single pass
- `canonical_token_value(path, home, token)` â€” converts a custom token's raw path value to its canonical storage form (`{{SYNC_HOME}}/Dev` for `/Users/bradmatic/Dev`)
- `fields::WHITELISTED` â€” 7 entries per spec Â§4.3; use `apply_to_whitelisted(obj, fn)` for L2 and `apply_to_all_strings(value, fn)` for L3
- `L3_WARNING` const in `levels.rs` â€” must be emitted by CLI commands (US-013, US-014, US-017) at startup when `level >= 3`
- proptest in `levels.rs` tests: import `use proptest::prelude::*;` inside the `mod tests` block (not at crate root)


- `TokenRegistry` in `src/canon/mod.rs` â€” built via `from_config(&CanonicalizationConfig, &Path)`; exposes `canonicalize_pi_dir`, `canonicalize_claude_dir`, `decanonicalize_pi_dir`, `decanonicalize_claude_dir`
- Canonicalization order: SYNC_HOME first, custom tokens after (applied to already-canonical form using canonical-encoded token values); de-canon reverses the order
- `boundary_match(s, prefix)` helper: character after prefix in `s` must be `-` or end-of-string; prevents partial matches (e.g., `/Users/bradmatic2` â‰  `/Users/bradmatic`)
- Custom tokens sorted by descending value-path length at construction time so most-specific prefix wins; only the first match applies (prefix-only replacement, so `break` after first hit is correct)
- Canonical encoded form of a custom token value: encode the raw path with agent encoding, then apply SYNC_HOME prefix replacement to that encoded string â€” this is the pattern matched against the already-canonicalized directory inner string
- `pi_encode_inner` / `claude_encode_inner` are module-private helpers; prefer `canonicalize_*_dir` / `decanonicalize_*_dir` in callers
- `Agent` trait in `src/agents/mod.rs` â€” `session_dir`, `encode_dir`, `decode_dir`; `PiAgent` and `ClaudeAgent` are unit structs implementing it
- Pi directory encoding: strip leading `/`, replace `/` with `-`, wrap with `--`; decoding is lossless only for components without `-`
- Claude directory encoding: strip leading `/`, replace `/` and `.` with `-`, prefix with `-`; decoding is lossless only for components without `-` or `.`
- Claude `decode_dir` explicitly rejects `--`-prefixed (Pi-format) names to catch cross-agent mistakes
- `cargo fmt` must be run (not just `--check`) after `Write` tool outputs code; rustfmt reformats `let x = name.strip_prefix(...)` differently from `let x = name\n    .strip_prefix(...)`
- Config module uses `pub(crate)` helpers (`apply_env_overrides`, `apply_cli_overrides`, `config_path_with_xdg_home`) to test precedence logic without mutating `std::env` (which is unsafe in parallel tests)
- `write_all(b"raw bytes")` in tests instead of `writeln!(f, r#"..."#)` avoids `{{` / `}}` being consumed as escaped format braces by the `writeln!` macro
- Separate `PiAgentConfig` / `ClaudeAgentConfig` structs needed because a shared `AgentConfig` can't have per-agent `session_dir` defaults with serde; distinct types each have their own `#[serde(default = "...")]` functions
- `#[derive(Default)]` on `Config` works after removing manual `impl Default` that just called `field::default()` â€” clippy::derivable_impls catches these
