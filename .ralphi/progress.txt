## Codebase Patterns
- RingBuffer path isolation: use `RingBuffer::path_for_repo(&repo_path)` in all _impl functions, NOT `RingBuffer::default_path()`. Mirrors the StateCache pattern.
- Temp filename uniqueness: use `as_nanos()` (full nanoseconds since epoch, u128) not `subsec_nanos()` (only the sub-second part, u32). Ring buffer already used this pattern; StateCache was the outlier.
- `git add -A` in this repo picks up untracked files outside src/ — prefer `git add src/ tests/ .ralphi/` or check `git status` before committing with `--no-edit`.

- Serde validation pattern: use `#[serde(default = "fn", deserialize_with = "fn")]` on a field to combine default-on-missing with a custom range/value validator at parse time. The default function bypasses the deserialize_with path (only used when field is absent), so the default value must itself be valid.
- CLI set_config_value pattern: always add explicit range/enum guards after `.parse::<T>()` for fields that have constrained valid sets.
- Tests for constrained config fields: add boundary tests in both `config/schema.rs` mod tests (TOML parse path) and `cli/mod.rs` mod tests (CLI set path).
