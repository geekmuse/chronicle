## Codebase Patterns
- **Scheduler section pattern**: split into `emit_scheduler_from_status(fmt, &ScheduleStatus)` (pure, testable, no I/O) and `emit_scheduler_section(fmt)` (calls `crontab_read()` then delegates). Unit tests inject synthetic `ScheduleStatus` values â€” no crontab process spawned.
- **next_run_secs helper**: `fn next_run_secs(interval_name: &str) -> Option<u64>` â€” parses `Xm`/`Xh` suffix, computes `interval_secs - (now_secs % interval_secs)`. Returns `None` for unknown formats; callers omit 'next run in' clause gracefully.
- **Verbose section pattern**: accept `verbose: bool` in emit functions; collect paths/values during normal processing then print indented extras in a post-block `if verbose { ... }`. Avoid a second scan pass.
- **Porcelain completeness**: emit all spec Â§3.3 keys in every run. On config error, emit the remaining keys with empty values (`fmt.kv("key", "")?`) so scripts always receive the full key set.
- **fmt.writer direct access**: `writeln!(fmt.writer, "  ...")` is valid from within functions that have a `&mut StatusFormatter<W>`. `fmt.writer` is a public field. Used for verbose indented output lines that don't carry symbols.
- **emit_* section pattern**: each status section is a private `fn emit_<name>_section<W: io::Write>(fmt, ...) -> io::Result<()>`. Wired in `status_write` after config is loaded. Errors within a section use `fmt.warn/err` (never propagate) â€” status always succeeds.
- **Read-only status rule**: `emit_pending_files_section` and `emit_lock_state_section` are strictly read-only; they load state files but never write them.
- **Lock state logic**: lock file format is `"<PID> <UNIX_TIMESTAMP>"`. Stale = `pid_dead || age_exceeded`. Use `.is_some_and(|p| !is_process_alive(p))` for dead-PID check (clippy prefers `is_some_and` over `map_or(false, ...)`). `#[allow(clippy::cast_sign_loss)]` on a `let` statement (not on an expression â€” that is a nightly-only feature).
- **SystemTime â†’ DateTime<Utc>**: use `(std::time::UNIX_EPOCH + std::time::Duration::from_secs(ts)).into()` to convert a u64 Unix timestamp â€” avoids extra chrono imports and is free of deprecation warnings.
- **Status formatter pattern**: `StatusFormatter<W: io::Write>` takes a generic writer for testability. Tests pass `&mut Vec<u8>` to capture output. `ok/warn/err` are no-ops in porcelain mode; `kv` is a no-op outside porcelain mode. Color codes (ANSI_GREEN/YELLOW/RED/RESET constants) applied to the symbol only.
- **should_use_color helper**: `pub fn should_use_color(no_color_flag: bool) -> bool` â€” checks flag, then `NO_COLOR` env var, then `io::stdout().is_terminal()`. Marked `#[must_use]`.
- **Private inner write function**: `status_write<W: io::Write>(args, config_path, home, use_color, writer)` is private but accessible from `mod tests` (child modules can access parent private items in Rust). Allows output capture without spawning a process.
- **Config re-export**: `config::Config` is available directly (via `pub use schema::Config` in `config/mod.rs`). No need to spell out `config::schema::Config` in function signatures.
- **Atomic file write pattern**: serialize to `.<name>.<pid>.<nanos>.tmp` sibling, then `fs::rename` to target. Used by `StateCache`, `MaterializeCache`, and now `SyncState`. Always call `fs::create_dir_all(parent)` first.
- **Co-location convention**: state files sit as siblings of the repo dir (`<repo_path>/../<file>.json`). The `path_for_repo(repo_path)` method returns `repo_path.parent().unwrap_or(repo_path).join("<file>.json")`.
- **Non-fatal side-effect writes**: wrap state file writes in `if let Err(e) = ... { tracing::warn!(...) }` â€” never fail the main operation on bookkeeping errors.
- **Operation timing**: capture `std::time::Instant::now()` AFTER lock acquisition (before real work), use `.elapsed()` at end for accurate duration excluding jitter.
- **Module registration**: new modules must be added to `src/lib.rs` as `pub mod <name>;` so integration tests can import them.
