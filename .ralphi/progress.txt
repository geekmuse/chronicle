## Codebase Patterns
- Doctor check functions go in `src/doctor/mod.rs` (extracted from cli/mod.rs because check logic exceeds ~300 lines).
- All doctor check functions accept injected paths/state to enable unit testing without real FS/network.
  - Pass a closure `impl Fn(&str) -> Result<(), String>` for network checks (remote reachability).
  - Pass `&[PathBuf]` for SSH key paths, `&[String]` for crontab lines, `&Path` for lock file.
- `CheckState` / `CheckResult` constructors (`pass`, `warn`, `error`, `skipped`) keep each call site concise.
- Use `#[must_use]` on all public check functions that return `Vec<CheckResult>`.
- The pre-existing flaky test `push_merges_new_entries_with_existing_repo_content` fails under parallel
  test execution (advisory lock contention). Run `cargo test -- --test-threads=1` for a reliable full run.
- `cargo deny check` emits warnings for unused license allowances and duplicate `getrandom` versions;
  these are pre-existing and not errors.
