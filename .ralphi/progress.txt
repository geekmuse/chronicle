## Codebase Patterns
- `sync_impl` and `pull_impl` both have a fast-path guard: skip
  `materialize_repo_to_local` when `integrate_remote_changes` returns 0.
  Always mirror this guard in both functions when touching materialize logic.
- Tests that call `pull_impl` (non-dry-run, with remote) need a bare git repo
  created with `git2::RepositoryInitOptions::bare(true)` and `write_push_config`.
  The repo can then be pre-populated by calling
  `git::RepoManager::init_or_open` + manual fs writes before invoking pull_impl.
- `integrate_remote_changes` returns 0 immediately when the remote tracking
  ref (`refs/remotes/origin/<branch>`) does not exist (empty remote).
- Cache structs (`StateCache`, `MaterializeCache`) live in separate modules and
  share the same API: `load(path)`, `save(path)`, `path_for_repo(repo_path)`.
  `path_for_repo` always returns a sibling of the repo directory (one level up).
  Atomic write uses `.mcache.<pid>.<nanos>.tmp` then `fs::rename` — copy this
  pattern exactly for any new cache types.
- The pre-commit hook (`ralphi check`) takes ~60s; use a 180s bash timeout when
  committing from within an agent session.
