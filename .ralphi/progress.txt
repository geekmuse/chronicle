## Codebase Patterns
- `TokenRegistry` in `src/canon/mod.rs` ‚Äî built via `from_config(&CanonicalizationConfig, &Path)`; exposes `canonicalize_pi_dir`, `canonicalize_claude_dir`, `decanonicalize_pi_dir`, `decanonicalize_claude_dir`
- Canonicalization order: SYNC_HOME first, custom tokens after (applied to already-canonical form using canonical-encoded token values); de-canon reverses the order
- `boundary_match(s, prefix)` helper: character after prefix in `s` must be `-` or end-of-string; prevents partial matches (e.g., `/Users/bradmatic2` ‚âÝ `/Users/bradmatic`)
- Custom tokens sorted by descending value-path length at construction time so most-specific prefix wins; only the first match applies (prefix-only replacement, so `break` after first hit is correct)
- Canonical encoded form of a custom token value: encode the raw path with agent encoding, then apply SYNC_HOME prefix replacement to that encoded string ‚Äî this is the pattern matched against the already-canonicalized directory inner string
- `pi_encode_inner` / `claude_encode_inner` are module-private helpers; prefer `canonicalize_*_dir` / `decanonicalize_*_dir` in callers
- `Agent` trait in `src/agents/mod.rs` ‚Äî `session_dir`, `encode_dir`, `decode_dir`; `PiAgent` and `ClaudeAgent` are unit structs implementing it
- Pi directory encoding: strip leading `/`, replace `/` with `-`, wrap with `--`; decoding is lossless only for components without `-`
- Claude directory encoding: strip leading `/`, replace `/` and `.` with `-`, prefix with `-`; decoding is lossless only for components without `-` or `.`
- Claude `decode_dir` explicitly rejects `--`-prefixed (Pi-format) names to catch cross-agent mistakes
- `cargo fmt` must be run (not just `--check`) after `Write` tool outputs code; rustfmt reformats `let x = name.strip_prefix(...)` differently from `let x = name\n    .strip_prefix(...)`
- Config module uses `pub(crate)` helpers (`apply_env_overrides`, `apply_cli_overrides`, `config_path_with_xdg_home`) to test precedence logic without mutating `std::env` (which is unsafe in parallel tests)
- `write_all(b"raw bytes")` in tests instead of `writeln!(f, r#"..."#)` avoids `{{` / `}}` being consumed as escaped format braces by the `writeln!` macro
- Separate `PiAgentConfig` / `ClaudeAgentConfig` structs needed because a shared `AgentConfig` can't have per-agent `session_dir` defaults with serde; distinct types each have their own `#[serde(default = "...")]` functions
- `#[derive(Default)]` on `Config` works after removing manual `impl Default` that just called `field::default()` ‚Äî clippy::derivable_impls catches these
- `HistoryMode` enum uses `#[serde(rename_all = "lowercase")]` + `#[default]` on `Partial` variant for clean TOML deserialization
- `cargo check` is fast after first compile; `cargo test` recompiles a separate test binary
- ICU unicode crates (icu_normalizer_data, icu_properties_data) come in via git2‚Üíurl‚Üíidna‚Üíicu_normalizer; first compile takes ~10min, cached afterwards
- git2 pinned to 0.18 (not 0.19/0.20) to avoid icu_normalizer_data in the dependency chain
- Dead-code for foundational error types: use `#![allow(dead_code)]` at the module level with a comment pointing to the stories that will wire it in
- All cli handler stubs live in `src/cli/mod.rs`; main.rs only owns the clap struct and match dispatch
