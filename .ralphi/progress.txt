## Codebase Patterns
- `sync_impl(dry_run, quiet, config_path, home)` ‚Äî testable core of `handle_sync`; follows same injection pattern as push_impl/pull_impl
- `build_updated_manifest(manager, machine_name, now)` ‚Äî reads current manifest, upserts `last_sync` for machine (creates entry with `first_seen=now` if absent), returns updated struct without writing; caller writes to disk (+optionally stages)
- Sync manifest strategy: write manifest.json to disk always; stage+commit it only when outgoing session changes exist (combined in the same commit); on idempotent sync (no outgoing changes) manifest is written to disk only, no separate commit
- Idempotent materialize: `materialize_agent_dir` checks `fs::read_to_string(local_file)` == `decanon_content` before writing; equal content skips write and does NOT increment total (satisfies "no file writes" on repeated syncs)
- HEAD guard before push: `if manager.repository().head().is_ok()` ‚Äî skips push on repos with no commits yet (edge case: fresh repo with no session data + empty remote)
- Sync git exchange skipped when `remote_url.is_none()` ‚Äî logs "[git] No remote configured" instead of failing; ring_buf created only inside the remote block

- `push_impl(dry_run, config_path, home)` / `pull_impl(dry_run, config_path, home)` ‚Äî testable cores of `handle_push` / `handle_pull`; inject home so tests don't touch real `$HOME`
- `ScannedChange` struct bundles `entry`, `source_dir`, `repo_rel_base`, `is_pi` ‚Äî collects scan results before deciding dry-run vs live path
- `PushFileParams<'a>` / `PushedFile` ‚Äî param bundle + result struct for `process_push_file`; keeps function under 7 arguments (clippy::too_many_arguments)
- `process_push_file`: L1 dir canon ‚Üí L2/L3 content canon ‚Üí merge with repo version if exists ‚Üí write; returns `Ok(None)` when merged result == repo content (nothing to stage)
- `integrate_remote_changes(manager, machine_name)`: walks `refs/remotes/origin/main` (fallback `master`) tree via git2 tree walk collecting `(path, git2::Oid)` pairs, then resolves blobs; merges each changed JSONL file; stages + commits if anything changed
- `materialize_repo_to_local(repo_path, cfg, registry)` / `materialize_agent_dir(...)`: walk repo session subdirs, `decanonicalize_*_dir` on each subdir name, `decanonicalize_line` per line, `write_preserving_permissions`
- `write_preserving_permissions(dest, content)`: `#[cfg(unix)]` captures existing mode before write; new files use parent-dir mode `& 0o666` or `0o644`; non-unix just does `fs::write`
- `non_empty_machine_name(name)`: returns `"unknown"` if empty; used as fallback in push/pull
- `git2` crate usable directly in `cli/mod.rs` (direct Cargo dep); no `use git2` import needed ‚Äî use full qualified paths like `git2::TreeWalkMode::PreOrder`
- Push retry: `push_with_retry("origin", || Ok(()), std::thread::sleep)` ‚Äî `on_rejection` is no-op for push-only (no fetch-merge cycle); exhaustion logged to ring buffer via `errors::ring_buffer::RingBuffer`
- Pull fetch error handling: `Err(ref e) if git::is_network_error(e)` logs to ring buffer before returning; other fetch errors propagate directly
- Remote tracking ref: try `refs/remotes/origin/main` then `refs/remotes/origin/master`; if neither exists ‚Üí `Ok(0)` (no remote commits yet)
- `git2::TreeWalkResult::Ok` is the return value needed from the tree walk closure (not `()`)


- `cli::import_impl(agent, dry_run, config_path, home)` ‚Äî testable core of `handle_import`; accepts injected home dir so tests don't touch real `$HOME`
- `ImportParams<'a>` struct in `cli/mod.rs` ‚Äî bundles all per-agent-import params to avoid clippy::too_many_arguments (>7 args)
- `manager: Option<&git::RepoManager>` in `ImportParams` ‚Äî `None` in dry-run mode; `import_agent_sessions` calls `.expect()` before using it (only reached in non-dry-run path)
- Import dry-run pattern: skip `RepoManager::init_or_open` entirely when `dry_run=true`; no repo path is created
- `import_agent_sessions` loop: collect + sort subdirs, skip dirs with no `.jsonl` files, for each dir write all files then `stage_files` + `commit_if_staged` (one commit per dir)
- L1 canon in import: `registry.canonicalize_pi_dir(dir_name)` / `canonicalize_claude_dir(dir_name)` on the source subdir name; result is the repo working-tree subdir name
- Content canon in import: `registry.canonicalize_line(line, canon_level)` per line; `unwrap_or_else` falls back to original line with `eprintln!` warning on error
- Trailing newline preservation: `if raw.ends_with('\n') && !canonical_content.ends_with('\n') { canonical_content.push('\n'); }`
- Staged path strings: `format!("{repo_rel_base}/{canonical_dir}/{filename}")` ‚Üí `PathBuf::from(...)` ‚Üí deref to `&Path` for `stage_files`
- `git::RepoManager`: wraps `git2::Repository`; `init_or_open(path, remote_url)` ‚Üí open if exists (ErrorCode::NotFound triggers init), set remote if provided; `ensure_working_tree()` creates pi/sessions/ + claude/projects/ (both with .gitkeep) + .chronicle/; `ensure_manifest()` writes default if absent; all public methods are idempotent
- `git::GitError`: thiserror enum ‚Äî `Git2(git2::Error)`, `Io { path, source }`, `Manifest(String)`; private `GitError::io(path, source)` constructor reduces boilerplate
- `git::Manifest` / `git::MachineEntry`: serde structs for `.chronicle/manifest.json`; `version: u32` (always 1), `machines: HashMap<String, MachineEntry>`; `last_sync` is `Option<DateTime<Utc>>` with `skip_serializing_if = "Option::is_none"`
- `open_or_init(path)`: module-private helper; `Repository::open` ‚Üí if `ErrorCode::NotFound` ‚Üí `create_dir_all` + `Repository::init`; other errors propagated
- `set_remote(name, url)`: private method; `find_remote` ‚Üí if NotFound add, else update URL via `remote_set_url`
- `.gitkeep` files placed in `pi/sessions/` and `claude/projects/` so git tracks empty dirs; `.chronicle/` left without .gitkeep (manifest.json serves that purpose)
- `serde_json::to_string_pretty` for manifest writes (human-readable JSON)
- `scan::StateCache`: `files: HashMap<String, FileState>`; `load(path)` (missing ‚Üí empty), `save(path)` (atomic temp+rename), `default_path()` (`~/.local/share/chronicle/state.json`); key is canonical repo path in production, absolute local path in unit tests
- `scan::FileState`: `local_mtime: DateTime<Utc>`, `local_size: u64`, `last_synced_size: u64`, `local_path: PathBuf`; serialized with `skip_serializing_if` is NOT needed here (all fields required)
- `scan::scan_dir(dir, cache, follow_symlinks)` ‚Üí `Result<Vec<ScanEntry>, ScanError>`; checks symlink on `dir` via `fs::symlink_metadata`; walks via `collect_jsonl` (skips inner symlinks); classifies via `classify(path, mtime, size, cache)` using abs-path key
- `scan::ChangeKind`: `New` (key absent), `Modified` (key present, mtime or size differs), `Unchanged` (both match)
- `scan::ScanError`: `SymlinkRefused(PathBuf)` | `Io { path, source }`
- `DateTime::<Utc>::from(system_time)` ‚Äî idiomatic `SystemTime ‚Üí DateTime<Utc>` conversion; `Utc.timestamp_opt(secs, nsecs)` needs `use chrono::TimeZone;` in the test mod
- `std::io::Error::other(e)` ‚Äî prefer over `std::io::Error::new(ErrorKind::Other, e)` (clippy::io_other_error fires on the latter)
- `collect_jsonl` uses `DirEntry::file_type()` (lstat, no symlink follow) and skips `ft.is_symlink()` entries ‚Äî only the top-level `dir` symlink is policy-checked

- `merge::entry`: `EntryKey` enum (`Header` | `Entry{entry_type,id}`); `parse_entry(raw)` returns `Option<ParsedEntry>`; `extract_timestamp(value)` checks `timestamp`, `created_at`, `createdAt` fields
- `merge::set_union::merge_jsonl(remote, remote_path, local, local_path, reporter)` ‚Üí `MergeOutput { content, malformed, conflicts }`; remote entries populate first (win on collision); prefix verification (¬ß5.4) detects byte-inequivalent common entries
- `ConflictReporter` trait in `merge::set_union` ‚Äî `report_prefix_mismatch(file, entry_key, local_raw, remote_raw)`; `NullReporter` is the no-op impl; real ring buffer impl comes in US-008
- `PrefixConflict { file, entry_key, local_raw, remote_raw }` ‚Äî one record per divergent common entry; included in `MergeOutput::conflicts`
- Prefix verification loop: iterate local_entries; if key in remote map AND raws differ ‚Üí warn + reporter.report + push conflict; remote version stays in map (skip local); if key not in remote map ‚Üí insert local
- `MalformedLine { path, line_number, snippet }` ‚Äî 1-based line, up to 80-char snippet; emitted via `tracing::warn!` and returned in `MergeOutput::malformed`
- Stable sort tie-break: `Source::Remote = 0`, `Source::Local = 1`, then `original_index` ‚Äî guarantees deterministic output for equal timestamps
- `Path::new("x")` is NOT const-stable in current Rust edition; use `fn helper() -> &'static Path { Path::new("x") }` in test constants
- `#![allow(dead_code)]` needed on both `entry.rs` and `set_union.rs` until US-007/US-015 wire in the merge module
- `canonicalize_line(line, level)` / `decanonicalize_line(line)` on `TokenRegistry` ‚Äî content-level L2/L3 dispatch in `src/canon/levels.rs`; returns `Result<String, ChronicleError>`
- `try_canonicalize_path(s)` ‚Äî L2: returns `Some(canonical)` only if `s == $HOME` or `s.starts_with($HOME/)` (strict prefix, boundary `/` or EOS); custom tokens applied prefix-only with `break`
- `try_canonicalize_text(s)` ‚Äî L3: `replace_in_text` scans globally; custom tokens also applied globally (no `break`)
- `try_decanonicalize_text(s)` ‚Äî reverses both L2 and L3; always applied to ALL string values in de-canon (safe because token pattern unlikely to appear naturally)
- `replace_in_text(text, from, to)` ‚Äî path-boundary replacement: `from` must be followed by `/` or EOS; prevents partial matches (e.g., `bradmatic2` won't match `bradmatic`); handles multiple occurrences in a single pass
- `canonical_token_value(path, home, token)` ‚Äî converts a custom token's raw path value to its canonical storage form (`{{SYNC_HOME}}/Dev` for `/Users/bradmatic/Dev`)
- `fields::WHITELISTED` ‚Äî 7 entries per spec ¬ß4.3; use `apply_to_whitelisted(obj, fn)` for L2 and `apply_to_all_strings(value, fn)` for L3
- `L3_WARNING` const in `levels.rs` ‚Äî must be emitted by CLI commands (US-013, US-014, US-017) at startup when `level >= 3`
- proptest in `levels.rs` tests: import `use proptest::prelude::*;` inside the `mod tests` block (not at crate root)


- `TokenRegistry` in `src/canon/mod.rs` ‚Äî built via `from_config(&CanonicalizationConfig, &Path)`; exposes `canonicalize_pi_dir`, `canonicalize_claude_dir`, `decanonicalize_pi_dir`, `decanonicalize_claude_dir`

- `boundary_match(s, prefix)` helper: character after prefix in `s` must be `-` or end-of-string; prevents partial matches (e.g., `/Users/bradmatic2` ‚âÝ `/Users/bradmatic`)

- `pi_encode_inner` / `claude_encode_inner` are module-private helpers; prefer `canonicalize_*_dir` / `decanonicalize_*_dir` in callers
- `Agent` trait in `src/agents/mod.rs` ‚Äî `session_dir`, `encode_dir`, `decode_dir`; `PiAgent` and `ClaudeAgent` are unit structs implementing it
- Pi directory encoding: strip leading `/`, replace `/` with `-`, wrap with `--`; decoding is lossless only for components without `-`

- Claude `decode_dir` explicitly rejects `--`-prefixed (Pi-format) names to catch cross-agent mistakes
- `cargo fmt` must be run (not just `--check`) after `Write` tool outputs code; rustfmt reformats `let x = name.strip_prefix(...)` differently from `let x = name\n    .strip_prefix(...)`
- Config module uses `pub(crate)` helpers (`apply_env_overrides`, `apply_cli_overrides`, `config_path_with_xdg_home`) to test precedence logic without mutating `std::env` (which is unsafe in parallel tests)

- Separate `PiAgentConfig` / `ClaudeAgentConfig` structs needed because a shared `AgentConfig` can't have per-agent `session_dir` defaults with serde; distinct types each have their own `#[serde(default = "...")]` functions
- `#[derive(Default)]` on `Config` works after removing manual `impl Default` that just called `field::default()` ‚Äî clippy::derivable_impls catches these
